`expandTilde` (port verbatim from upstream `bin/install.js:479-484`):
```javascript
function expandTilde(p) {
  if (p && p.startsWith('~/')) return path.join(os.homedir(), p.slice(2));
  return p;
}
```

`resolveConfigDir(adapter, parsed, env)` algorithm (D-03):
1. If `parsed.configDir` truthy → return it (already tilde-expanded + path.resolve'd in parseCliArgs)
2. Else if `env[adapter.configDirEnvVar]` truthy → return `expandTilde(env[adapter.configDirEnvVar])`
3. Else → return `path.join(os.homedir(), adapter.defaultConfigDirSegment)`

`detectPresentRuntimes(homeDir = os.homedir())` (port from RESEARCH.md Example 2, lines 619-628):
```javascript
const SUPPORTED_RUNTIMES = ['claude', 'codex', 'gemini'];
const DEFAULT_SEGMENTS = { claude: '.claude', codex: '.codex', gemini: '.gemini' };
function detectPresentRuntimes(homeDir = os.homedir()) {
  const present = [];
  for (const runtime of SUPPORTED_RUNTIMES) {
    const dir = path.join(homeDir, DEFAULT_SEGMENTS[runtime]);
    try { if (fs.statSync(dir).isDirectory()) present.push(runtime); } catch { /* ENOENT */ }
  }
  return present;
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI argv → bin/lib/args.cjs | User-supplied flags including `--config-dir <path>` enter the process from argv |
| process.env → bin/lib/args.cjs::resolveConfigDir | Environment variables (`CLAUDE_CONFIG_DIR` etc.) read but not written |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | T (Tampering) — path traversal | `parseCliArgs` `--config-dir` value | mitigate | `path.resolve(expandTilde(value))` normalizes `..` segments. The resolved path is passed to subsequent `fs.cp` / `fs.mkdir` calls (Plan 04 / 06). The args module itself does not write to disk; downstream callers assume the path is honest. Path-traversal mitigation lives at the writer in Plan 04 / 06 (must refuse to write outside `configDir`). |
| T-03-02-02 | T — tilde expansion ambiguity | `expandTilde` | mitigate | Only `~/` prefix is expanded; `~user` (other user's home) is intentionally NOT expanded — POSIX-only POSIX-tuned per CONTEXT.md "Out of Scope: Windows support" + RESEARCH.md Pitfall context. Ambiguity resolved by spec. |
| T-03-02-03 | I (Information disclosure) | env var read | accept | Reading `CLAUDE_CONFIG_DIR`/`CODEX_HOME`/`GEMINI_CONFIG_DIR` is the documented contract (ADR-02 do-not-rename allowlist). Values logged in `--verbose` could expose user paths to stdout — acceptable for personal-use tool. |
| T-03-02-04 | D (DoS) | parseArgs strict mode | mitigate | `strict: true` rejects unknown flags. Malformed argv produces `ArgError` (exit 3), no infinite loop. |
| T-03-02-05 | E (Elevation of privilege) | `runtime-detect.cjs::detectPresentRuntimes` | mitigate | TOCTOU window between `statSync` (detect) and later `fs.cp` (write) is benign for personal use — no shared filesystem. If symlink exists at `~/.claude` pointing elsewhere, `statSync(...).isDirectory()` follows it (Node default); writes go where the symlink points. Documented as out-of-scope per CONTEXT.md (clean-slate, single-user). |
</threat_model>
