`injectMarkerBlock(filePath, openMarker, closeMarker, body)` — three-case algorithm port from upstream `mergeCopilotInstructions` at `foundation-frameworks/get-shit-done-main/bin/install.js:3349-3380` (RESEARCH.md Pattern 4, lines 339-364):
```javascript
function injectMarkerBlock(filePath, openMarker, closeMarker, body) {
  const block = `${openMarker}\n${body.trim()}\n${closeMarker}`;
  if (!fs.existsSync(filePath)) {
    fs.writeFileSync(filePath, block + '\n');
    return;
  }
  const existing = fs.readFileSync(filePath, 'utf8');
  const openIdx = existing.indexOf(openMarker);
  const closeIdx = existing.indexOf(closeMarker);
  if (openIdx !== -1 && closeIdx !== -1) {
    const before = existing.substring(0, openIdx).trimEnd();
    const after  = existing.substring(closeIdx + closeMarker.length).trimStart();
    let newContent = '';
    if (before) newContent += before + '\n\n';
    newContent += block;
    if (after) newContent += '\n\n' + after;
    fs.writeFileSync(filePath, newContent + '\n');
    return;
  }
  fs.writeFileSync(filePath, existing.trimEnd() + '\n\n' + block + '\n');
}
```

`removeMarkerBlock(filePath, openMarker, closeMarker)` — ported from upstream `stripGsdFromCopilotInstructions` at `bin/install.js:3388-3402`:
```javascript
function removeMarkerBlock(filePath, openMarker, closeMarker) {
  if (!fs.existsSync(filePath)) return;
  const content = fs.readFileSync(filePath, 'utf8');
  const openIdx = content.indexOf(openMarker);
  const closeIdx = content.indexOf(closeMarker);
  if (openIdx === -1 || closeIdx === -1) return;
  const before = content.substring(0, openIdx).trimEnd();
  const after  = content.substring(closeIdx + closeMarker.length).trimStart();
  const cleaned = (before + (before && after ? '\n\n' : '') + after).trim();
  if (!cleaned) fs.unlinkSync(filePath);
  else fs.writeFileSync(filePath, cleaned + '\n');
}
```

`findUpstreamMarkers(filePath)` — D-24 detection (RESEARCH.md lines 379-387):
```javascript
function findUpstreamMarkers(filePath) {
  if (!fs.existsSync(filePath)) return [];
  const content = fs.readFileSync(filePath, 'utf8');
  const markers = [];
  if (content.includes('<!-- GSD Configuration')) markers.push('GSD');
  if (content.includes('<EXTREMELY_IMPORTANT>You have superpowers.')) markers.push('Superpowers');
  if (content.match(/superpowers-codex bootstrap/i)) markers.push('Superpowers-Codex');
  return markers;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied instruction file → `marker.cjs` | Existing CLAUDE.md/AGENTS.md/GEMINI.md content read & rewritten — must preserve user content outside markers |
| state file (`<configDir>/oto/.install.json`) → `install-state.cjs::readState` | JSON deserialization of disk content; could be tampered between install/uninstall sessions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | T (Tampering) — marker injection | `marker.cjs::injectMarkerBlock` | mitigate | Markers are static literal strings (`<!-- OTO Configuration -->` / `<!-- /OTO Configuration -->`) — never user-supplied. The `body` argument comes from `adapter.renderInstructionBlock(ctx)` which itself is static (renders `oto v${OTO_VERSION}` from package.json — not user input). No path traversal because `filePath` is computed by orchestrator as `path.join(configDir, adapter.instructionFilename)` where `instructionFilename` is a hardcoded descriptor field. |
| T-03-03-02 | T — marker spoofing | `marker.cjs::injectMarkerBlock` over hostile pre-existing content | mitigate | If user has placed counterfeit `<!-- OTO Configuration -->` markers in their CLAUDE.md, our injection will overwrite between them. This is the intended idempotent behavior; user-controlled content outside markers preserved. Acceptable for personal-use single-user scope. |
| T-03-03-03 | I (Information disclosure) — state file content | `install-state.cjs::writeState` | mitigate | Pitfall H explicitly forbids absolute paths in `state.files[].path` — `validateState` rejects with error message including `'must be relative to configDir'` so debug pastes don't leak `$HOME`. `state.config_dir` is absolute (necessary for uninstall reconstruction) but is the only absolute path. No secrets in state. |
| T-03-03-04 | T — corrupt-state recovery | `install-state.cjs::readState` | mitigate | JSON.parse failure → wrapped Error 'state file corrupt:'. Schema validation failure → wrapped Error 'state file invalid:'. Both bubble up and the orchestrator (Plan 06) treats these as fatal install errors (exit 1 / 2). Does NOT silently treat corrupt as missing — that would cause shadowed re-installs. |
| T-03-03-05 | D (DoS) — large instruction file | `marker.cjs` `fs.readFileSync` | accept | If user has a multi-GB CLAUDE.md, sync read blocks the event loop. Personal-use ceiling — out-of-scope concern. Documented in CONTEXT.md (no concurrent-install lock; single user). |
| T-03-03-06 | E (Elevation) — symlink at instruction file path | `marker.cjs::injectMarkerBlock` writing through symlink | accept | If `~/.claude/CLAUDE.md` is a symlink to `/etc/passwd`, our `fs.writeFileSync` follows it. Mitigated by file ownership (write only succeeds with user's perms) and the personal-use single-user assumption. Out-of-scope for v0.1.0 (CLAUDE.md "What NOT to Use" notes Windows symlink issues; POSIX symlinks within $HOME are user's own files). |
| T-03-03-07 | S (Spoofing) — fake schema version | `install-state.cjs::validateState` | mitigate | Strict equality check (`state.version !== CURRENT_SCHEMA_VERSION`); rejects forward-version (e.g., a Phase 5+ state file read by a Phase 3 installer). Forces explicit migration when schema rev'd. |
</threat_model>
