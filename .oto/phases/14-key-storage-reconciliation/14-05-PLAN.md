| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-14-05-01 | Information Disclosure | buildNewProjectConfig / configNewProject write | mitigate | reconcileNewProjectIntegrations validates the fully merged object before atomicWrite; post-merge boolean gate in SDK |
| T-14-05-02 | Information Disclosure | rejection error message | mitigate | reuse validateIntegrationValue message which never echoes the submitted value; tests assert marker absent from all output |
| T-14-05-03 | Tampering | keyfile-wins conflict during defaults migration | mitigate | existing keyfile is never overwritten by a defaults string (D-02); masked notice only |
| T-14-05-04 | Elevation of Privilege | keyfile creation during migration | mitigate | writeKeyfile enforces dir 0700 + file 0600 + chmod; test asserts (mode & 0o777) === 0o600 |
| T-14-05-05 | Repudiation | silent defaults rewrite | accept | best-effort defaults.json heal is try/catch-silent by design (matches existing depth→granularity precedent); failure only causes repeat masked notices |
