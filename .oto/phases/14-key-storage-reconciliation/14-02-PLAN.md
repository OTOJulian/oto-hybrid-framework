## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| oto-sdk CLI argv → configSet | key strings arriving as query arguments |
| SDK process → git-tracked .oto/config.json | committed file; anything written is exposed |
| SDK process → ~/.oto keyfiles | secret material at rest |
| SDK process → ~/.gsd/ (foreign install) | belongs to the separate GSD install; must not be read or written |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-09 | I | configSet persisting a key string to tracked config.json (live defect — SDK path has no masking at all) | mitigate | validateIntegrationValue throws Validation before lock/write (D-05) |
| T-14-10 | I | legacy string read repeatedly without healing | mitigate | migrateLegacyIntegrationKeys hooked in configGet AND loadConfig (D-01) |
| T-14-11 | T | SDK reading/writing ~/.gsd/ keyfiles (wrong trust domain, foreign install) | mitigate | D-08 hard cutover to ~/.oto with test proving .gsd files are ignored; no fallback |
| T-14-12 | I | keyfile created world-readable | mitigate | mode 0o600 at write + unconditional chmod + read heal (D-12) |
| T-14-13 | I | plaintext key in GSDError messages / stderr notices | mitigate | messages carry masked values only; test asserts stderr lacks plaintext |
| T-14-14 | T | concurrent config writes corrupting config.json | mitigate | existing acquireStateLock + atomicWriteConfig retained; migration uses temp+rename |
| T-14-15 | D | migration exception breaking every configGet/loadConfig | mitigate | try/catch guards at both hook sites |
