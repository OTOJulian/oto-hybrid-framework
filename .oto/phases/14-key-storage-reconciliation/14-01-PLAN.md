## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell argv → oto-tools CLI | key strings arriving as command arguments (already exposed to shell history/ps) |
| CJS process → git-tracked .oto/config.json | anything written here is committed and pushed |
| CJS process → ~/.oto keyfiles | secret material at rest on disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | I (Info disclosure) | cmdConfigSet writing key string to tracked config.json | mitigate | validateIntegrationValue hard-reject before any write (D-05); nothing persisted anywhere |
| T-14-02 | I | legacy key string already sitting in tracked config.json | mitigate | migrateLegacyIntegrationKeys on every CJS config load moves it to 0600 keyfile (D-01) + git-history rotation advice (D-03) |
| T-14-03 | I | keyfile readable by other local users | mitigate | write mode 0o600 + chmod after write + read-time permission heal (D-12) |
| T-14-04 | I | key echoed in CLI stdout/stderr or test logs | mitigate | all notices use maskSecret (`****<last-4>`); tests assert plaintext absent from output |
| T-14-05 | T (Tampering) | partial config rewrite corrupting config.json during migration | mitigate | temp-file + rename atomic write in secrets.cjs |
| T-14-06 | T | path traversal via integration slug into arbitrary file write | mitigate | INTEGRATIONS allowlist — keyfilePath throws on unknown slug |
| T-14-07 | S (Spoofing) | attacker-planted keyfile impersonating user's key | accept | local single-user machine; keyfile dir is 0700 under $HOME |
| T-14-08 | D (DoS) | migration failure blocking every config load | mitigate | try/catch wrapper in loadConfig hook; migration itself no-ops on unreadable config |
</threat_model>
