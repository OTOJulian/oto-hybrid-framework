Classification rules (D-09/D-11/D-14):
- runtime config dir absent → `not-installed`
- no live entry + no state record → `not-registered`
- live entry + record, equal → `oto-managed`
- live entry + record, differ → `drifted`
- live entry + NO record → `user-owned`  (codex: an external `[mcp_servers.exa]` outside the OTO block is also `user-owned`, even alongside an oto block)
- record + no live entry → `missing-but-expected`

Coherence warnings (D-10) — pure function over gathered inputs: (a) `exa_search: true` in project config but zero runtimes registered → warn; (b) any runtime registered (oto-managed/drifted) but `detectKeySource('exa').source === null` → warn.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted config files → status parser | Status live-parses user-editable JSON/TOML; malformed content must not crash or mislead |
| Status output → user decisions | A wrong "oto-managed" claim could lead the user to delete an entry they own |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-29 | Denial of Service | malformed config/TOML crashing status or doctor | mitigate | All parses in try/catch; unparseable → tolerated with detail flag; matrix includes corrupt-input cases |
| T-15-30 | Spoofing | user-owned entry misclassified as oto-managed (enabling wrongful deletion advice) | mitigate | Classification requires fingerprint deep-equality against the state record; classifier never mutates anything (read-only by construction) |
| T-15-31 | Information Disclosure | key material in status output | mitigate | Status consumes only `detectKeySource(...).source`; output strings are static + paths; no masked-value display needed here |
| T-15-32 | Repudiation | status resolution diverging from registration resolution (FRESH-CR-02 regression) | mitigate | Same env-var-else-default rules, claude via the same `claudeJsonPath`; env-override tests pin parity |
</threat_model>
