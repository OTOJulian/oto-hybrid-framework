## Shared mergeMcp/unmergeMcp adapter contract (identical wording in Plans 04/05/06 â€” Plan 07's dispatcher relies on it)

```
async mergeMcp(ctx) -> { registered: boolean, refused: null | { reason: string, existing?: any },
                         entry: <fingerprint unit> | null, target: <absolute path written/inspected> }
ctx = { configDir, env, otoVersion, launcherPath, priorEntry }
  // priorEntry = fingerprint recorded in .install.json state.mcp.exa.entry from a prior install, or null

async unmergeMcp(ctx) -> { removed: boolean, skipped: null | { reason: 'absent' | 'user-owned' | 'drifted' }, target }
ctx = { configDir, env, otoVersion, priorEntry }
```
Merge decision matrix (D-13 / MCP-07):
- no existing exa entry â†’ write, `registered: true`, `entry` = what was written
- existing entry matches priorEntry OR matches what oto would write â†’ rewrite in place (idempotent refresh), `registered: true`
- existing entry, no match â†’ `refused: { reason: 'user-owned' }`, file untouched
Unmerge decision matrix (D-14 / MCP-08):
- no live entry â†’ `{ removed: false, skipped: { reason: 'absent' } }`
- live entry equals priorEntry â†’ remove it, `{ removed: true }`
- live entry differs from priorEntry (or priorEntry null) â†’ skip, reason 'drifted' (or 'user-owned'), one-line stderr: `oto: exa entry was modified since oto registered it â€” left in place.`

For Codex the fingerprint unit is the block's INNER content string (lines between the MCP markers, exclusive).
