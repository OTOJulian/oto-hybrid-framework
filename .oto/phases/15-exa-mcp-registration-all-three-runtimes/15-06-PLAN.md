## Shared mergeMcp/unmergeMcp adapter contract (identical wording in Plans 04/05/06 — Plan 07's dispatcher relies on it)

```
async mergeMcp(ctx) -> { registered: boolean, refused: null | { reason: string, existing?: any },
                         entry: <fingerprint unit> | null, target: <absolute path written/inspected> }
ctx = { configDir, env, otoVersion, launcherPath, priorEntry }

async unmergeMcp(ctx) -> { removed: boolean, skipped: null | { reason: 'absent' | 'user-owned' | 'drifted' }, target }
ctx = { configDir, env, otoVersion, priorEntry }
```
Merge decision matrix (D-13 / MCP-07):
- no existing exa entry → write, `registered: true`, `entry` = what was written
- existing entry matches priorEntry OR matches what oto would write → rewrite in place (idempotent refresh), `registered: true`
- existing entry, no match → `refused: { reason: 'user-owned' }`, file untouched
Unmerge decision matrix (D-14 / MCP-08):
- no live entry → `{ removed: false, skipped: { reason: 'absent' } }`
- live entry equals priorEntry → remove it, `{ removed: true }`
- live entry differs (or priorEntry null) → skip, reason 'drifted' / 'user-owned', one stderr line: `oto: exa entry was modified since oto registered it — left in place.`

Equality = deep-equal via key-sorted stable stringify (same ~8-line helper as Plan 05 — inline a local copy; Wave-2 plans must not share files).
