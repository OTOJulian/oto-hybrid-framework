```yaml
# Byte-cloned from foundation-frameworks/get-shit-done-main/.github/workflows/install-smoke.yml
# (smoke-tarball + smoke-unpacked job pattern). Adjustments from upstream:
#  - binary name: gsd-sdk → oto (oto ships raw .js, no SDK build step)
#  - removed upstream `chmod 644 sdk/dist/cli.js` step: oto has no tsc-fresh
#    artifacts; bin/install.js is checked in at mode 755 in the repo. Per
#    W-2 fix and T-INST-01 mitigation update, npm preserves source mode on
#    unpacked installs, so the smoke validates "fresh clone install works",
#    which is the actual user-facing scenario.
name: Install Smoke

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
    paths:
      - 'bin/**'
      - 'package.json'
      - 'scripts/install-smoke.cjs'
      - '.github/workflows/install-smoke.yml'
      - '.github/workflows/release.yml'
  workflow_call:
  workflow_dispatch:

concurrency:
  group: install-smoke-${{ github.workflow }}-${{ github.head_ref || github.run_id }}
  cancel-in-progress: true

jobs:
  smoke-tarball:
    runs-on: ${{ matrix.os }}
    timeout-minutes: 12
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: ubuntu-latest
            node-version: 22
          - os: ubuntu-latest
            node-version: 24
          - os: macos-latest
            node-version: 24
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd  # v6.0.2
      - uses: actions/setup-node@53b83947a5a98c8d113130e565377fae1a50d02f  # v6.3.0
        with:
          node-version: ${{ matrix.node-version }}
      - name: Ensure npm global bin is on PATH
        shell: bash
        run: |
          NPM_BIN="$(npm config get prefix)/bin"
          echo "$NPM_BIN" >> "$GITHUB_PATH"
          echo "npm global bin: $NPM_BIN"
      - name: Pack and install tarball
        shell: bash
        run: |
          set -euo pipefail
          TARBALL=$(npm pack --silent)
          TMPDIR_ROOT=$(mktemp -d)
          cd "$TMPDIR_ROOT"
          npm install -g "$GITHUB_WORKSPACE/$TARBALL"
          command -v oto
          oto --help || oto
      - name: Assert oto is executable
        shell: bash
        run: |
          set -euo pipefail
          OTO_PATH="$(command -v oto)"
          test -x "$OTO_PATH"
          echo "✓ oto resolves at: $OTO_PATH (executable)"

  # Mirrors upstream smoke-unpacked job: install from $GITHUB_WORKSPACE
  # without npm pack. Catches the failure class where source files in the
  # repo would lose execute bits (mode-644 trap, npm/cli #2453). Source
  # bin/install.js is mode 755 in the repo; this job proves npm preserves it.
  smoke-unpacked:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd  # v6.0.2
      - uses: actions/setup-node@53b83947a5a98c8d113130e565377fae1a50d02f  # v6.3.0
        with:
          node-version: 22
      - name: Ensure npm global bin is on PATH
        shell: bash
        run: |
          NPM_BIN="$(npm config get prefix)/bin"
          echo "$NPM_BIN" >> "$GITHUB_PATH"
          echo "npm global bin: $NPM_BIN"
      - name: Install from unpacked workspace (no npm pack)
        shell: bash
        run: |
          set -euo pipefail
          TMPDIR_ROOT=$(mktemp -d)
          cd "$TMPDIR_ROOT"
          npm install -g "$GITHUB_WORKSPACE"
          command -v oto
          oto --help || oto
      - name: Assert oto is executable after unpacked install (#2453 regression check)
        shell: bash
        run: |
          set -euo pipefail
          OTO_PATH="$(command -v oto)"
          test -x "$OTO_PATH"
          echo "✓ oto is executable after unpacked install: $OTO_PATH"
```

name: Release

  cancel-in-progress: false

jobs:
  validate:
    runs-on: ubuntu-latest
    timeout-minutes: 5
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd  # v6.0.2

        shell: bash
        run: |
          set -euo pipefail
          TAG="${GITHUB_REF_NAME#v}"
          PKG=$(node -p "require('./package.json').version")
          if [ "$TAG" != "$PKG" ]; then
            echo "::error::tag v$TAG does not match package.json $PKG"
            exit 1
          fi

  test:
    needs: validate
    uses: ./.github/workflows/test.yml

  smoke:
    needs: validate
    uses: ./.github/workflows/install-smoke.yml

  release:
    needs: [test, smoke]
    runs-on: ubuntu-latest

    permissions:

# Upstream Sync

oto is a downstream fork of two upstreams: [Get Shit Done (GSD)](https://github.com/gsd-build/get-shit-done) and [Superpowers](https://github.com/obra/superpowers). New commits in either upstream are pulled, rebranded, diffed against oto, and surfaced as conflicts for manual resolution.

```sh

## How sync works

1. **Pull** — `scripts/sync-upstream/pull-gsd.cjs` and `pull-superpowers.cjs` fetch the latest upstream `main` (or pinned tag) and write a snapshot to `.oto-sync/upstream/{gsd,superpowers}/`.
2. **Rebrand** — `scripts/sync-upstream/rebrand.cjs` applies `rename-map.json` to the snapshot, producing a rebranded tree.
3. **Diff** — `scripts/sync-upstream/merge.cjs` compares the rebranded tree against the current `oto/` and emits any conflicts to `.oto-sync-conflicts/<path>.md`.
4. **Surface** — Per-upstream `BREAKING-CHANGES.md` log captures upstream removals/renames that need oto-side action.
5. **Accept / keep** — Operator resolves each conflict manually via `oto sync --accept` (take upstream) or `oto sync --keep-deleted` (preserve oto-local copy of upstream-deleted file).

Each conflict file in `.oto-sync-conflicts/` follows a header schema (Phase 9 D-12) describing the upstream → oto delta. After hand-merging, run `oto sync --accept <path>` to mark resolved and update `last-synced-commit.json`.

# Rebrand Engine

oto is built by transforming GSD and Superpowers source through a rule-typed rebrand engine that converts identifiers, paths, slash commands, skill namespaces, package names, URLs, and env vars per a declarative `rename-map.json`. The engine is what makes oto a true fork rather than a wrapper.

## Why rule-typed?

A naive `s/gsd/oto/g` corrupts URLs, copyright lines, license text, and any `gsd` substring inside an unrelated identifier (e.g. `magsd`). The engine instead routes each match through one of seven rule classes that know exactly what they're allowed to touch:

## Rule classes

| Rule class | Pattern shape | Example |
|-----------|----------------|---------|
| `identifier` | `\b`-bounded word | `gsdConfig` → `otoConfig` |
| `path` | path-segment match | `.gsd/STATE.md` → `.oto/STATE.md` |
| `command` | `/<prefix>-<slug>` | `/gsd-plan-phase` → `/oto-plan-phase` |
| `skill_ns` | `<ns>:<skill>` | `Skill('superpowers:tdd')` → `Skill('oto:test-driven-development')` |
| `package` | npm package name | `"name": "gsd-build"` → `"name": "oto"` |
| `url` | URL host/path | `github.com/gsd-build/get-shit-done` (preserved in attribution context) |
| `env_var` | `^[A-Z_]+$` | `GSD_HOME` → `OTO_HOME` |

Each rule class lives in its own module under [`scripts/rebrand/lib/rules/`](../scripts/rebrand/lib/rules/).
