**Bug 2 — agent map lacks inherit branch.** `sdk/src/query/config-query.ts:59-67` falls back to `balanced` for unknown profile keys. CJS reference (`model-profiles.cjs:87-95`):
```javascript
function getAgentToModelMapForProfile(normalizedProfile) {
  const agentToModelMap = {};
  for (const [agent, profileToModelMap] of Object.entries(MODEL_PROFILES)) {
    agentToModelMap[agent] = normalizedProfile === 'inherit'
      ? 'inherit'
      : profileToModelMap[normalizedProfile];
  }
  return agentToModelMap;
}
```

**Bug 3 — init.quick resolves 'sonnet' instead of 'inherit'.** Confirmed cause: this repo's `.oto/config.json` has BOTH `"model_profile": "inherit"` AND `"resolve_model_ids": "omit"` (line 39). In `resolveModel` (`config-query.ts:152-194`) the omit check (line 174) runs BEFORE the inherit check (line 188), returning `model: ''`. Then `getModelAlias` in `sdk/src/query/init.ts:39-43` does `(data.model as string) || 'sonnet'`, coercing `''` → `'sonnet'`. Fix: check `profile === 'inherit'` immediately after the per-agent `model_overrides` check (which stays highest precedence), before the omit check — per `oto/references/model-profile-resolution.md:31`: "If model_profile is 'inherit', all agents resolve to 'inherit'". No change needed in init.ts once resolveModel returns 'inherit'.
