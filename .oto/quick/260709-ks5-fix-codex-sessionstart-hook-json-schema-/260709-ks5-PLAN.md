<!-- Confirmed this session against openai/codex@main codex-rs/hooks/src/schema.rs.
     Field lists below are the ONLY fields deny_unknown_fields will accept (all optional
     unless noted). Anything else on the JSON top level fails the whole hook parse. -->

SessionStartCommandOutputWire (camelCase, deny_unknown_fields):
  universal fields (flattened, all optional): continue, stopReason, suppressOutput, systemMessage
  hookSpecificOutput?: SessionStartHookSpecificOutputWire { hookEventName, additionalContext }

PreToolUseHookSpecificOutputWire (nested under hookSpecificOutput, all fields optional except hookEventName):
  hookEventName, permissionDecision?, permissionDecisionReason?, updatedInput?, additionalContext?

PostToolUseHookSpecificOutputWire (nested under hookSpecificOutput):
  hookEventName, additionalContext?, updatedMcpToolOutput?

Exit-code contract (PreToolUse and PostToolUse, confirmed from codex-rs/hooks/src/events/{pre,post}_tool_use.rs):
  exit 0  -> stdout parsed as JSON (empty stdout = success, no-op)
  exit 2  -> tool call blocked; reason comes from stderr (raw text, stdout NOT parsed as JSON)
  other   -> hook run treated as failed

Current runtime-detection cascade in oto/hooks/oto-session-start (bash), branch order matters:
```bash
if [ -n "${CURSOR_PLUGIN_ROOT:-}" ]; then
  printf '{\n  "additional_context": "%s"\n}\n' "$session_context"
elif [ -n "${CLAUDE_PLUGIN_ROOT:-}" ] && [ -z "${COPILOT_CLI:-}" ]; then
  printf '{\n  "hookSpecificOutput": {\n    "hookEventName": "SessionStart",\n    "additionalContext": "%s"\n  }\n}\n' "$session_context"
else
  printf '{\n  "additionalContext": "%s"\n}\n' "$session_context"
fi
```
