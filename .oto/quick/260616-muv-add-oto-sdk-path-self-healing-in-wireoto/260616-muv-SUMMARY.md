## Objective

Root-cause prevention for `command not found: oto-sdk` after the framework repo
is relocated on disk. `wireOtoSdk()` writes an absolute symlink
(`<repoRoot>/bin/oto-sdk.js`) into a HOME-owned PATH dir; a move dangles it.

## What shipped

### Task 1 — self-heal the dangling managed symlink (`62336ea`)
- `bin/lib/install.cjs`: `isManagedOtoSdkTarget()` now handles the dangling-symlink
  case. When `realpathSync` throws (target missing), it falls back to a raw
  `readlinkSync` and treats a link whose basename is `oto-sdk.js` as a
  stale-managed link → returns `true`. Previously it returned `false`, so
  `trySelfLinkOtoSdk()` skipped the candidate dir and never recreated the link.
  Net effect: a relocated repo self-heals on the next `oto install`.
- Exported `isManagedOtoSdkTarget` and `findOtoSdkOnPath` from `module.exports`
  (consumed by `doctor.cjs`).
- `tests/sdk-wiring.test.cjs`: +3 cases (dangling managed → true; dangling
  unmanaged → false; `trySelfLinkOtoSdk` heals a dangling managed link). 11 pass.

### Task 2 — `oto doctor` subcommand (`8bf672c` RED, `164a219` GREEN)
- New `bin/lib/doctor.cjs`: `checkOtoSdk(opts)` returns a structured verdict —
  `healthy` | `missing` | `stale` (`reason: dangling | wrong-repo`) | `shadowed`,
  plus `sdkDistMissing` when the shim is correct but `sdk/dist/cli.js` is absent.
  Reuses `findOtoSdkOnPath` / `isManagedOtoSdkTarget`. `main()` prints actionable
  remediation and exits 0 only when healthy; Windows degrades gracefully.
- **Dangling-vs-missing nuance resolved via option (a):** because
  `findOtoSdkOnPath` uses `statSync` (follows links) a dangling on-PATH link
  would naively read as `missing`. `checkOtoSdk` adds a secondary `lstat` scan of
  PATH dirs so a dangling managed link reports `stale`/`dangling` with the better
  "re-run install" message instead of a bare `missing`.
- `bin/install.js`: `argv[0] === 'doctor'` dispatch (mirrors `sync`/`migrate`/`log`)
  + `HELP_TEXT` entry.
- `tests/260616-muv-doctor.test.cjs`: 5 verdict tests (healthy, healthy+dist-missing,
  missing, stale/wrong-repo, shadowed). All pass.

## Notes / follow-ups
- The self-heal triggers on the next `oto install`; it does not run on every
  command invocation. `oto doctor` is the on-demand check between installs.
- Detection is basename-heuristic (`oto-sdk.js`), only ever acting on HOME-owned
  PATH dirs (existing `path.startsWith(home + sep)` guard) — no new attack surface.
