## Milestone Summary

### Key Accomplishments

- Shipped `/oto-migrate` as a project-level converter that walks an external GSD-era project tree, classifies planning artifacts and instruction-file marker blocks, applies the rebrand engine in dry-run by default, and supports idempotent re-runs with timestamped backups and half-migrated conflict detection.
- Shipped `/oto-log` as a fire-and-forget and session-bookmarked ad-hoc work capture surface, with evidence-grounded six-section body drafting, prompt-injection guardrails (`<DATA_START>`/`<DATA_END>`), immutable `.oto/logs/` entries, and `list` / `show` / `promote --to quick|todo` subcommands.
- Wired `/oto-log` into `/oto-progress` Recent Activity (interleaved with phase summaries chronologically) and `/oto-resume-work` (active-session detection via `.oto/logs/.active-session.json`).
- Added per-runtime parity for both new commands across Claude Code, Codex, and Gemini, with generated runtime tool matrix entries showing PASS for all three.
- Hardened `/oto-migrate` post-release with four follow-up quick fixes (Codex skill adapter parity, runtime worktree exclusion, public CLI dispatch, gitignored-artifact skipping).

### Key Decisions

- `.planning/` → `.oto/` directory rename in `/oto-migrate` is **opt-in** (`--rename-state-dir`), not default. Default preserves the directory name and only rewrites contents — minimizes blast radius for users with CI/hooks that grep `.planning/`, and matches oto-this-repo's actual behavior.
- `/oto-migrate` is a **thin orchestrator** that delegates in-file string transforms to the existing `scripts/rebrand/lib/engine.cjs`. New code is limited to detection, scope-globbing, marker-block rewrite, frontmatter-key rewrite, backup/restore, and conflict detection.
- `/oto-log` body is drafted from **observable evidence only** (recent transcript + `git diff` + `git log` over a bounded ref). Diff content is wrapped in `<DATA_START>`/`<DATA_END>` markers and capped at 8 KB; injected DATA marker literals in diff bytes are escaped before wrapping.
- `/oto-log` entries are **immutable**. No `edit` subcommand. Same-minute collisions get `-2`/`-3` suffixes via `fs.openSync('wx')` exclusive-create with retry.
- `/oto-log promote` supports `--to quick|todo` only. `--to plan` is explicitly rejected before any file read or write side effects.
- Phase 01 introduced `safeJoin`, `assertNotRuntimeConfigDir`, and a `MIGRATE_SKIP_PATTERNS` set that excludes runtime agent worktrees, `.git`, `node_modules`, and `.oto-migrate-backup` from migrate's walk.
- v0.1.1 was a small interim tag for Codex parity fixes (nested `[[hooks.<Event>]]` schema + `~/.codex/skills/oto-<name>/SKILL.md` + `$oto-<name>` invocation). v0.2.0 supersedes it.
