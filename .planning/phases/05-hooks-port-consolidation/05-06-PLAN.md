<interfaces>
`oto-validate-commit.sh` remains opt-in. If `.oto/config.json` is missing or `hooks.session_state !== true`, it exits 0 without reading `.oto/STATE.md`.

When opt-in is enabled and the command is a `git commit` form, enforce this order:
1. The commit message must be parseable from `-m`, `--message`, or `--message=`.
2. The subject must pass the existing Conventional Commits checks.
3. `.oto/STATE.md` must exist.
4. `.oto/STATE.md` must contain an active phase line and an active plan line.

Use simple, deterministic text checks for the state invariant:
- active phase: a line beginning `Phase:` whose value does not contain `Not started`, `none active`, `COMPLETE`, or `completed`.
- active plan: a line beginning `Plan:` whose value is not empty and not `—`, `-`, `N/A`, or `Not started`.

Block failures with JSON on stdout and exit 2. Use reasons containing the exact substrings `active phase` and `active plan` so tests can assert the invariant.
</interfaces>
</context>
