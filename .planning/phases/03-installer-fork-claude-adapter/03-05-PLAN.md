<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `package.json` version → adapter `renderInstructionBlock` | The oto version is read by the orchestrator from `package.json` (trusted source) and passed through `ctx.otoVersion` |
| Adapter source code → user instruction file | Marker block content is composed by adapter code (trusted, in repo) — not user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | T (Tampering) — marker injection via crafted version string | `renderInstructionBlock` interpolates `ctx.otoVersion` | mitigate | `ctx.otoVersion` originates from `package.json` `version` field, set at release time and write-controlled by repo maintainer. Never user-supplied. Even if an attacker tampered with `package.json` post-install, the marker is bracketed by static `OPEN_MARKER`/`CLOSE_MARKER` so an injected `</marker>X<!-- ` payload couldn't escape the block — the closing marker is appended by `marker.cjs::injectMarkerBlock`, not the adapter. |
| T-03-05-02 | I (Information disclosure) — marker content | `renderInstructionBlock` output | accept | Block content is intentionally public: oto version, repo URL, command list pointer. No user paths, no env vars, no secrets. Stable content per CONTEXT.md "Specifics" line 331 (snapshot tested). |
| T-03-05-03 | T (Tampering) — Phase 3 stub transforms accidentally non-identity | identity transforms in Codex/Gemini adapters | mitigate | Tests assert literal identity behavior (`transformCommand('foo', {}) === 'foo'`). TODO-marker grep tests assert the explicit `// TODO Phase 8` comments are present so future devs know Phase 3 is intentionally a stub, not a forgotten obligation. |
| T-03-05-04 | E (Elevation) — `onPreInstall` reading instruction file | `findUpstreamMarkers` via `fs.readFileSync` | mitigate | `findUpstreamMarkers` (Plan 03) returns `[]` if file absent. The path it reads is `path.join(ctx.configDir, adapter.instructionFilename)` where `instructionFilename` is a hardcoded descriptor field — no path traversal. The function only reads (no write), so no privilege escalation. |
| T-03-05-05 | S (Spoofing) — wrong adapter dispatch | adapter.name field used by orchestrator for dispatch | mitigate | Orchestrator (Plan 06) selects adapter by string match on parsed runtime flag — there's no untrusted source for the dispatch. The adapter cannot lie about its `name` field because the orchestrator picks the adapter module by file path, not by the `name` field. The `name` field is for logging only. |
</threat_model>
