|----------|-------------|
| test process → tmp filesystem | Test creates and writes under `os.tmpdir()`; cleanup is mandatory to avoid filling `/tmp` over CI runs |
| test process → oto-tools.cjs subprocess | Spawns lib subprocess with controlled env; trust boundary is the env itself |
| concurrent Claude instances → session-keyed pointers | The behavior under test IS the mitigation for this STRIDE-T threat (T-07-03 in Plan 01) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08 | Tampering | tmp dir leakage if test crashes mid-subtest | mitigate | Use `t.after(...)` for cleanup so it runs even on assertion failure; OS will eventually GC `os.tmpdir()` anyway |
| T-07-09 | Spoofing | session-key collision (hardcoded test session keys could collide with real sessions) | accept | Test runs in a tmp dir; session-keyed pointers are stored under tmp/.oto/.session-pointers/ — no path collision with real user `.oto/` |
| T-07-10 | Tampering | inheriting OTO_* env vars from outer shell pollutes test | mitigate | Each spawnSync call passes a curated env: `{ ...process.env, OTO_WORKSTREAM: '', OTO_SESSION_KEY: 'fixture-key-N', ... }` overriding any ambient values |
